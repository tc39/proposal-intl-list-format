1. Let _requestedLocales_ be ? CanonicalizeLocaleList(_locales_). 1. If _options_ is *undefined*, then 1. Let _options_ be ObjectCreate(*null*). 1. Else 1. Let _options_ be ? ToObject(_options_). 1. Let _s_ be GetOption(_options_, `"style"`, `"string"`, «`"long"`, `"short"`, `"narrow"`, `"long"`). 1. Set _listFormat_.[[Style]] to _s_. 1. Let _localeData_ be *%ListFormat%*.[[LocaleData]]. 1. DeconstructPattern ( _pattern_, _placeables_ )

The *DeconstructPattern* abstract operation is called with arguments _pattern_ (which must be a String) and _placeables_ (which must be a Record), and deconstructs the pattern string into a list of parts. Placeables record is a record whose keys are placeables tokens used in the pattern string, and values are parts records which will be used in the result List to represent the token part. Example: 

        Input:
          DeconstructPattern("AA{xx}BB{yy}CC", {
            xx: {Type: 'hour', Value: '15'},
            yy: {Type: 'minute', Value: '06'}
          });

        Output (List of parts records):
          [
            {Type: 'literal', Value: 'AA'},
            {Type: 'hour', Value: '15'},
            {Type: 'literal', Value: 'BB'},
            {Type: 'minute', Value: '06'},
            {Type: 'literal', Value: 'CC'}
          ]

1. Set _parts_ to be the result of _pattern_.split(/\{([^\}]+)\}/). 1. Let _result_ be a new empty List. 1. Set _i_ to be *0*. 1. Repeat, while _i_ < _parts_.length 1. Let _part_ to be _parts_.[_i_]. 1. If _i_ % 2 is *0*, then 1. If _part_.length is > *0*, then 1. Add new part record { [[Type]]: `"literal"`, [[Value]]: _part_ } as a new element on the list _result_. 1. Else, 1. If ? HasOwnProperty(_placeables_, _part_), then 1. Throw a TypeError exception.. 1. Set _subst_ to _placeables_.[[_part_]]. 1. If Type(_subst_) is a List, then 1. Let _len_ be ? ToLength(? Get(_subst_, `"length"`)). 1. Set _i_ to *0*. 1. Repeat, while _i_ < _len_ 1. Add _subst_.[_i_] as a new element on the list _result_. 1. Else, 1. Add _subst_ as a new element on the list _result_. 1. Return _result_.

The *CreatePartsFromList* abstract operation is called with arguments _listFormat_ (which must be an object initialized as a ListFormat) and _list_ (which must be a List), and creates the corresponding list of parts according to the effective locale and the formatting options of _listFormat_. Each part is a Record with two fields: Type, which must be a string with values "element" or "literal", and Value which must be a string or a number. The following steps are taken:

1. Let _len_ be the length of _list_. 1. If _list_ contains any element _value_ such that Type(_value_) is not String, throw a *TypeError* exception. 1. If _len_ is *0*, then 1. Return a new empty List. 1. If _len_ is *1*, then 1. Let _result_ be a new empty List. 1. Let _value_ be _list_[0]. 1. Add new part record { [[Type]]: `"element"`, [[Value]]: _value_ } as a new element of the list _result_. 1. Return _result_. 1. If _len_ is *2*, then 1. Let _pattern_ be _listFormat_.[[Template2]]. 1. Let _listValue1_ be _list_[0]. 1. Let _listValue2_ be _list_[1]. 1. Let _value_ be a new placeable record. 1. Set _value_.["0"] to be new part record { [[Type]]: `"element"`, [[Value]]: _listValue1_ }. 1. Set _value_.["1"] to be new part record { [[Type]]: `"element"`, [[Value]]: _listValue2_ }. 1. Return DeconstructPattern(_pattern_, _value_). 1. Let _v_ be _list_[_len_ - 1]. 1. Set _parts_ to be a new part record { [[Type]]: `"element"`, [[Value]]: _v_ }. 1. Set _i_ to _len_ - *2*. 1. Repeat, while _i_ > -1 1. If _i_ equals _len_ - *2*, then 1. Set _template_ to _listFormat_.[[TemplateEnd]]. 1. Else, if _i_ equals *0*, then 1. Set _template_ to _listFormat_.[[TemplateStart]]. 1. Else, 1. Set _template_ to _listFormat_.[[TemplateMiddle]]. 1. Let _value_ be a new placeable record. 1. Set _v_ to _list_[_i_]. 1. If Type(_v_) is not String, throw a *TypeError* exception. 1. Set _value_.["0"] to be new part record { [[Type]]: `"element"`, [[Value]]: _v_ }. 1. Set _value_.["1"] to be _parts_. 1. Set _parts_ to be the result of DeconstructPattern(_template_, _value_). 1. Set _i_ to _i_ - 1. 1. Return _parts_.

Internal slots

The value of the [[AvailableLocales]] internal slot is implementation defined within the constraints described in .

The value of the [[LocaleData]] internal slot is implementation defined within the constraints described in and the following additional constraints:

  • [[LocaleData]][Locale] must have a `formats` property for all locale values. The value of this property must be an object, which must have properties with the names of the three list formatting types: `"conjunction"`, `"disjunction"`, and `"unit"`. Each object must have properties with the names of three formatting styles: `"long"`, `"short"` and `"narrow"`. Each of those properties must be objects themselves, and each must have properties with names: `"2"`, `"start"`, `"middle"` and `"end"`. Each of those properties must be a template string as specified in LDML List Format Rules.
It is recommended that implementations use the locale data provided by the Common Locale Data Repository (available at http://cldr.unicode.org/). In LDML's listPattern, `"conjuction"` corresponds to "standard", `"disjuntion"` corresponds to "or", and `"unit"` corresponds to "unit". Among the list types, `"conjunction"` stands for "and"-based lists (e.g., "A, B and C"), `"disjunction"` stands for "or"-based lists (e.g., "A, B or C"), and `"unit"` stands for lists of values with units (e.g., "5 pounds, 12 ounces").

Properties of Intl.ListFormat Instances

Intl.ListFormat instances inherit properties from *%ListFormatPrototype%*.

Intl.ListFormat instances and other objects that have been successfully initialized as a ListFormat have [[initializedIntlObject]] and [[initializedListFormat]] internal slots whose values are *true*.

Objects that have been successfully initialized as a ListFormat object also have several internal slots that are computed by the constructor:

  • [[Locale]] is a String value with the language tag of the locale whose localization is used by the list format styles.
  • [[Type]] is one of the String values `"conjunction"`, `"disjunction"`, or `"unit"`, identifying the list of types used.
  • [[Style]] is one of the String values `"long"`, `"short"`, or `"narrow"`, identifying the list formatting style used.
